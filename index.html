What is Tumblr?

In the world of the web you have your big social networking platforms like Facebook or Twitter and then down the line you find a neat social network known as Tumblr which stands on its own.

Tumblr is a microblogging platform owned by Yahoo! Inc that can be used by pretty much anyone who knows how to operate a computer. Tumblr focuses on content first by allowing you, the author, to post virtually any type of media you want. Everything from stories, links, quotes, photos, GIFS, videos, audio, embeddable media, and more. The interface itself is iconic but still extremely easy to understand.

Users can follow each other similar to how you would on Twitter and also like each other’s blogs. The option also exists to make a blog private which I’ll teach you how to do later in this book. You can even redirect a tumblr blog to be accessible through your own custom domain name (ex.”www.mycustomtumblrtheme.com”) which makes your blog more personable to you or whatever content you decide to author.

At the time of this writing Tumblr hosts over 255 million blogs. That’s a substantial number that is continuously rising. More users are looking to blog about whatever it is they want and tumblr is a perfect tool for doing so.

Some users are interested in not only blogging but also using tumblr as their main website. This very concept is what lead for users to be able to customize their own themes to tailor fit their own use cases.

Stuff you should know first

I’m sure you are ready to get started, right? Well hold on, Before jumping in you should always do a bit of planning before beginning any project. First, you need to determine if Tumblr is indeed the right tool for what you require. While it’s a great blogging platform it does have it’s limitations. Some of these limitation’s we’ll discuss as we progress through this book but below I have listed some strengths and weaknesses of the platform that you should know before going forward.

Tumblr isn’t an end all be all tool but it can do a lot. You can:

  • Write blog posts containing virtually any type of media, even chats.
  • Author your own theme and colors
  • Add pages
  • Follow other users
  • Like other user’s posts
  • Make your blog private and accessible via password
  • Have your tumblr blog point to a custom domain you already own
  • Easily link and post to other popular social networks
  • Make your theme responsive (with the built in mobile app or
  • Choose from tons of pre-made themes (but why would you want to do that?)

Seems quite customizable aye? Well, unfortunately, you can’t

  • Touch any back-end code. This means you can’t change any of the core functionality when it comes to posting and administrative functionality.
  • Extend Tumblr into something more like popular CMS’s out there
  • Add third party plugins
  • Edit and preview your custom theme locally
  • Upload assets for your theme’s design without absolute URLs
  • Create more advanced websites with it (this is by design)
  • Host your tumblr blog yourself

Tumblr Source Code Structure

If you have coded a few websites or themes in the past you may know of various standards or best practices most developers stand by. We do this so that in the event we need to collaborate on a project the code involved is in an organized format.

Tumblr takes these practices and pretty much throws them out the window in some areas. The key differences between a tumblr blog and a traditional blog are noted below:

  • An entire theme is basically one HTML file (can be multiple)
  • CSS is typically embedded within the theme
  • All external resources and assets must be absolute URL paths
  • Get used to Tumblr variables as you’ll use theme everywhere
  • Hopefully, you are comfortable working inside a web-based code editor (kind of a pain in the ass)

Aside from these differences, the code that gets output is very similar to a blog of any type be it WordPress, Ghost, or Medium for example.

We’ll be getting our hands dirty inside this code editor going forward but for now, we need to get started on the design of the theme.

Designing a Custom Theme

Designing a custom tumblr theme can be done in a smart manner. Knowing what post types you’re going to support from the beginning will allow you to create a design that adapts to the specific style you are going for.

Blog Anatomy

A blog by nature is a feed of content that continuously grows based on date or other types of categorization. Whether you post one type of content or multiple this theory stands true. Blog posts are archived the older they get. This allows you to go back in time and reference a post you created years ago if you wish to.

Typically there is a header and footer area of a blog. These commonly display navigation or controls that help you traverse through a website. Headers and footers don’t always need to be top and bottom in terms of a website. While this is the most common form, the important thing is to keep these elements consistent between pages.

Luckily enough for us, a tumblr blog doesn’t have that many pages to content with unless you craft a custom page. I’ll talk more of custom pages coming up.

Having covered the header and footer leaves us with the main content area. Within this area will be the tumblr posts feed. Each post displays in a hierarchal format depending on how you define your settings. There are quite few post types to choose from when publishing content. I’ll dive deeper for each type but for now just know these possibilities include:

  • Reblogs
  • Text Posts
  • Photo Posts
  • Panorama Posts
  • Photoset(Gallery) Posts
  • Quote Posts
  • Link Posts
  • Chat Posts
  • Audio Posts
  • Video Posts
  • Answer Posts

Thinking Responsive

By default, tumblr deploys its own responsive theme for mobile and tablet based devices. You can toggle this setting in the options within the appearance screen. We are going to make use of our own design rather than employing tumblrs. This is to both be consistent as well as show you how it’s done.

The initial design I came up with for our theme doesn’t have a mobile or tablet-based design bundled with it. I did this on purpose because some of the design will take place in the browser. By this I mean, since we have our basic design defined, I can build upon or subtract from it when the time comes to support multiple screen widths.

Some would argue to design the mobile version first. While I like this concept I have found in my own methods for this to be less efficient. When getting to code I start with the desktop version and slowly make my way down to various screen sizes for optimal support. I find it easier to subtract than add in terms of what should be in view on mobile and what should be in view on the desktop version. This same idea applies to tablet devices or even really big screen sizes.

Design Tools

It’s worth mentioning that you don’t necessarily need to design a static design in software like Sketch or Photoshop in the first place. A lot of modern websites have been built from the ground up using just ideas and code.

Assets

Our theme doesn’t make use of a lot of raster images. In fact, the only assets we need to supply are some of the social icons and the user profile picture. I’ll be using SVGs for the icons.

A Quick background on SVG - SVGs (Scaleable Vector Graphics) are vector images that can simply scale. They are becoming more popular than images due to the large about of screen sizes and resolutions on the market these days. Combined with CSS, and JavaScript, SVGs can be modified in line. Color, animations, stroke settings and more can be configured depending on the vector image.

There’s a lot more on the topic and why SVGs are becoming so popular. I recommend checking out Chris’s blog post about them or watch Kyle’s SVG Workflow Video.

I use SVGs because they are scalable and customizable. If you’ve never used them before I recommend looking deeper into doing so. I’ll show you how to use a combination of SVG code and CSS to make some great effects in the future. What’s great about SVGs is that you can copy and paste the code inside your theme file directly. This means we won’t have to upload any assets which are a big perk while working on a Tumblr theme.

Developing a theme

How you develop your theme can vary. Tumblr has a built in code editor that works quite well but is a little limiting to me. I rarely like to code a real project in the browser even though it is possible these days. I use a code editor to start off my themes and virtually any other development type of project.

Using a native code editor gives me more freedom towards how I write my code, but unfortunately, I lose the ability to preview the markup Tumblr generates from its template tagging system locally. This setback results in me having to code the theme with placeholder content and imagery rather than plugging it into Tumblr from the start.

I’m personally okay with this because not a lot should change in the process. I’ve built quite a few themes before so I know what to expect going in. Hopefully, my tried and true methods speak to you so you don’t have to struggle like I did on my first few attempts.

Tumblr Blocks and Variables

Tumblr’s custom theme documentation does a great job of explaining quickly what each variable and block means as well as how to use it. I find it limiting in some areas but I certainly recommend reading further about any features you would like to support in your own theme. You will, without a doubt, be referencing it if you’re new to tumblr theme development. Each variable and block it supplies can open a lot of doors you didn’t know existed within the boundaries of your theme. Check out the list below and try to get acquainted. We will be making use of many of these and more shortly.

Blocks Overview

Blocks are written similar to an HTML tag with an opening and closing element, but they also tie into the Tumblr theming structure which happens behind the scenes. Blocks act as a containing element for content which gets presented in certain areas of the theme. Some common blocks we will be making use of are as follows

Page Related Blocks
{block:PermalinkPage}{/block:PermalinkPage}

Content within this block is rendered as the permalink page. This is a template for any post which is useful for embedding comment widgets or plugins.

{block:IndexPage}{/block:IndexPage}

This block is useful for defining a custom home or index page. If you need different content or want more customization you can use both the {IndexPage} block and the {PermalinkPage} block to achieve advanced results.

{block:PostTitle}{PostTitle}{/block:PostTitle}

Rendered on permalink pages. (Useful for displaying the current post’s title in the page title)

{block:HasPages}{/block:HasPages}

Tumblr will check to see if you created any pages and display them using the {HasPages} block.

{block:Pages}{/block:Pages}

Rendered for each custom page.

{block:PermalinkPagination}{/block:PermalinkPagination}

This block is useful for displaying pagination within permalink pages. This allows you to navigate to previous and next links in the post archive.

Post Related Blocks
{block:Posts}{/block:Posts}

This block gets rendered for each post in reverse chronological order. You can configure how many posts that get displayed inside the Customize area for the blog.

{block:Text}{/block:Text}

this block renders for text based posts.

{block:Photo}{/block:Photo}

this block renders for photo based posts.

{block:Panorama}{/block:Panorama}

If you have larger images to display you can use the panorama block to do so. These images can be massive if you wish to make them that large.

{block:Photoset}{/block:Photoset}

If a gallery is in order you can use the photoset block to achieve this. Tumblr does nothing more than group these photos together. You’ll need to implement some styling or JavaScript to create a realistic gallery effect.

{block:Quote}{/block:Quote}

this block renders for quote based posts.

{block:Link}{/block:Link}

this block renders for link based posts.

{block:Chat}{/block:Chat}

this block renders for chat based posts.

{block:Audio}{/block:Audio}

this block renders for audio based posts. Audio can be embedded using a widget or uploaded directly.

{block:Video}{/block:Video}

this block renders for video based posts. Video can be embedded using a widget or uploaded directly.

{block:Answer}{/block:Answer}

Tumblr makes use of an Ask feature which can be switched on within the Customize area. If switched on, you can choose to author a post based on a question that way asked from one of your blog readers. This block renders that content.

{block:RebloggedFrom} {/block:RebloggedFrom}

Think of re-blogging like sharing on Facebook. You can post another post using this block.

Variables Overview

Variables are just that, variables. These are used to generate dynamic content so you never have to go back and edit any HTML manually once your theme is all set up. Variables can be used within blocks and outside of blocks depending on their purpose. Some variables are global (usable anywhere), while others are relative to specific block types. The block types I’ve listed before correspond to specific variables which output a different type of data and values to make your theme authentic to your needs.

Common Global Variables

The Basics

{Title}

renders an HTML-safe title for your blog

{Description}
The description of your blog. (HTML can be included)

{MetaDescription}
Having a description inside your head of the HTML document is great for SEO. Use the {MetaDescription} variable to output this directly.

{BlogURL}

Often times you may want to link to your blog. This is easily done with the {BlogURL} variable

{RSS}

Giving your users an option to tie into an RSS feed is very handy. The {RSS} variable will render the appropriate link to the RSS feed for your blog.

{CustomCSS}

This gives you the option to add CSS inside the Customize section of the blog without having to edit any source code. This is useful for quick fixes or if you need the theme off to someone else who would rather figure things out on their own.

Appearance

These template tags/variables allow you to add customizable elements that revolve around your blog’s appearance. We’ll be using a handful of these for example purposes. These are completely optional. The default tumblr theme makes the most use of them.

{TitleFont}

The font used for your blog’s title

{BackgroundColor}

Set the background color of your blog dynamically.

{AccentColor}

Set the accent color of your blog dynamically. This is usually action items like links, buttons, and more.

{block:ShowHeaderImage}{/block:ShowHeaderImage}

This blog renders a header image if you have the option enabled in the customize area. If you want a big hero image for your theme then this is probably the easiest way to go about it.

{block:HideHeaderImage}{/block:HideHeaderImage}

If you disable the header image option the content within this block will show instead.

Adding metatags

The head tags in a Tumblr theme adds most of the dynamic properties a theme owner can configure to match their own personal brand.

Configurable options such as web typography, color palette, editable content areas, and more are added to a Tumblr theme through the use of meta tags.

When these are added between the head tags inside a theme a user will then be able to see an editable field when they click on the Edit Theme button similar as to what is displayed within the following image:

An example of this that we will make use of is a meta tag for Google Web Fonts. Our theme makes use of the web font called Oswald as you may have already noticed. We linked to the font’s CSS directly inside our theme using the following code:

<link href=’https://fonts.googleapis.com/css?family=Oswald:400,700,300’ rel=’stylesheet’ type=’text/css’>

To make this field editable, we need to first add a meta tag

<meta name=”text:Google Web Font name” content=“Oswald” />

And then modify our link tag from before to be:

<link href=’http://fonts.googleapis.com/css?family={text:Google Web Font name}’ rel=’stylesheet’ type=’text/css’>

By default without any user interaction the font will be set to Oswald because we passed that name through the content=“x” parameter.

Including a caption for a photo is as simple as adding the {block:Caption} tags and the {Caption} tag itself.

Panorama

Our panorama post html from before looks like this:


                <!-- Panorama Block Starts -->
                   <div class=”post panorama grid-item grid-item--panorama-w grid-item--panorama-h”>
                    <img src=”http://placekitten.com/g/850/324” alt=”{PhotoAlt}” />
                    <div class=”caption”>Lorem ipsum dolor sit amet, consectetur adipisicing elit.</div>
                  </div>
                <!-- Panorama Block Ends -->

Lets update it to tie into Tumblr.


                <!-- Panorama Block Starts -->
                 {block:Panorama}
                    <div class=”post panorama grid-item grid-item--panorama-w grid-item--panorama-h”>
                    {LinkOpenTag}<img src=”{PhotoURL-Panorama}” alt=”{PhotoAlt}” />{LinkCloseTag}
                    {block:Caption}<div class=”caption”>{Caption}</div>{/block:Caption}
                   </div>
                {/block:Panorama}
                <!-- Panorama Block Ends -->

The panorama> post type is very similar to the photo post type. You can even use variables from the photo post type within it.

The idea behind the panorama post type is to display large images in a different way than normal photos if you choose to do so. If your image is massive, Tumblr will automatically create a panorama post type from it when you’re adding a photo post.

Photoset

Photoset post types are generated when you attach more than one image to a photo post. They will display as a collection of which you can format however you please. In our design, we shrink up to four photos into a small rectangle. Clicking each will open a built in gallery component Tumblr offers by default.

Our original html looked like this:


                 <!-- Photoset Block Starts-->
                 <div class=”post photoset grid-item”>
                   <a href=”#”><img class=”photoset-photo” src=”http://placekitten.com/g/270/184” alt=”{PhotoAlt}” /></a>
                   <a href=”#”><img class=”photoset-photo” src=”http://placekitten.com/g/270/184” alt=”{PhotoAlt}” /></a>
                   <a href=”#”><img class=”photoset-photo” src=”http://placekitten.com/g/270/184” alt=”{PhotoAlt}” /></a>
                   <a href=”#”><img class=”photoset-photo” src=”http://placekitten.com/g/270/184” alt=”{PhotoAlt}” /></a>
                  <div class=”caption”>Lorem ipsum dolor sit amet, consectetur adipisicing elit.</div>
                 </div>
                <!-- Photoset Block Ends-->

As you can see I’m manually creating photos that are grouped in a block with the photoset class. To tie things into tumblr I altered the code below:


                 <!-- Photoset Block Starts-->
                 {block:Photoset}
                  	<div class=”post photoset grid-item”>
                   	 {block:Photos}
                     	 <div class=”photoset-photo”>
                     	 <img src=”{PhotoURL-250}” />
                     	 {block:Caption}<div class=”caption”>{Caption}</div>{/block:Caption}
                  	 </div>
                  	 {/block:Photos}
                	
                	 {block:Caption}<div class=”caption”>{Caption}</div>{/block:Caption}
                	
                	</div>
                 {/block:Photoset}
                <!-- Photoset Block Ends-->

Like all the other post types so far I have added the unique block identifier {block:Photoset} to communicate with Tumblr so that it knows where and how to post photo sets.

Inside the block identifier is another block labeled {Photos}. The content in between this block will be duplicated based on however many photos have been published. You can include the photo and a caption for each or you can include a global caption only for the photoset block.

We could go one step further and link the photoset photos to their larger versions using some sort of third-party library but I will leave that enhancement up to you.

Chat

If you need to display a conversation you can do so with chat post types. The html we started with looks like the following:


                <!-- Chat Block Starts-->
                <div class=”post chat grid-item”>
                  <h3><a href=”{Permalink}”>My Chat Title</a></h3>
                  <ul class=”chat”>
                    	<li class=”{Alt} user_{UserNumber}”>
                      	 <span class=”label”>Andy:</span>This is what Andy said
                    	</li>
                    	<li class=”{Alt} user_{UserNumber}”>
                      	 <span class=”label”>Reese:</span>This is what Reese said
                    	</li>
                    	<li class=”{Alt} user_{UserNumber}”>
                      	 <span class=”label”>Andy:</span>This is what Andy said again
                    	</li>
                  </ul>
                </div>
                <!-- Chat Block Ends-->

And our updated code:


                <!-- Chat Block Starts-->
                {block:Chat}
                 <div class=”post chat grid-item”>
                  		{block:Title}
                    			<h3><a href=”{Permalink}”>{Title}</a></h3>
                  		{/block:Title}

                 <ul class=”chat”>
                  {block:Lines}
                    <li class=”{Alt} user_{UserNumber}”>
                      <span class=”label”>{Name}:</span>{Line}
                    </li>
                  {/block:Lines}
                  </ul>
                 </div>
                {/block:Chat}
                <!-- Chat Block Ends-->

I defined the chat structure inside of an unordered list. Each list item will dynamically display the user and their chat excerpt. This is all housed inside the {block:Lines} tags.

I’ve added class names for each line with Tumblr tags. The {Alt} tag displays “odd” or “even” for each line of this post. The {UserNumber} tag gives each line a unique identifying integer representing the user of the current line of this post.

Permalink Pages

Permalink pages can be customized to look differently than posts on the index page. The goal of a permalink page is to single out a specific post and provide other functionality such as comments, sharing, reflagging, liking, and so on.

On the index page, I made only the text and photo post types capable of being clicked through to the permalink page. This is merely to show how a permalink page behaves as well as how to support it inside your own theme builds. You are more than welcome to extend the theme further to provide support for all post types.

Creating Pages

You can create new pages which display content unrelated to the native post like the structure of a tumblr blog. Having this functionality allows you to use Tumblr for a fully fledged website if you so desire.

To create a page head to the Customize area within the Tumblr dashboard and scroll down until you see the link + Add a page.

Clicking the link redirects you to a new window. From here you can name your page. Define what layout the new page will make use of by selecting the standard layout or custom layout and add the content using the WYSIWYG editor below. You also have the option to redirect to another link entirely.

Our page URLS will make use of the built-in tag structure so that when clicked will display all posts with the corresponding tag.

For example, I want to display all posts with the tag of Design. The URL we add to the page name will be webcrunchblog.tumblr.com/tagged/design.

Upon creating the page you should be able to navigate directly to it to see all of the posts published with the design tag.

Domains

If you want to make use of Tumblr and ditch the extension in the

domain name you can certainly do so. For this to take effect you will need to modify your domain provider’s A-record to point towards tumblr’s.

To edit your domain, head to your Blog Settings and look for the pencil icon to the right of your username.

Click on the pencil icon to reveal a new set of fields.

You can test your domain to see if it is pointing towards tumblr or not. If so you should be good to go. If not you will likely see an error

You will need to set the A-record for your domain to 66.6.44.4.

When changing any DNS record there is always some latency involved.

Be patient after making the switch. In my history of theme building, this has usually happened within an hour or less.

Responsiveness

Since we designed the site to be responsive we shouldn’t have any major tweaks to make to the code.

Clicking this link reveals some global options to tweak for your theme as well as an area to add custom CSS if your users would ever want to. We added support for this in our styles already.
